## 6. Risk Management

### Potential Risks & Mitigation

**Risk 1: Content Delays**
- **Impact:** Timeline pushed back
- **Mitigation:** Start content writing early (Week 2), have draft content ready
- **Contingency:** Use placeholder content, finalize later

**Risk 2: Technical Issues During Migration**
- **Impact:** Launch delay, downtime
- **Mitigation:** Keep old Webnode site live until new site fully tested
- **Contingency:** Can revert DNS back to old site if needed

**Risk 3: Client Approval Delays**
- **Impact:** Timeline extends
- **Mitigation:** Set clear approval deadlines, schedule review meetings
- **Contingency:** Build in 1-week buffer time

**Risk 4: Hosting/Technical Problems**
- **Impact:** Site downtime, performance issues
- **Mitigation:** Choose reliable host (Websupport.sk), test thoroughly
- **Contingency:** Have backup host option, maintain backups

**Risk 5: SEO Ranking Drop Post-Launch**
- **Impact:** Traffic decrease temporarily
- **Mitigation:** Proper 301 redirects, maintain URL structure where possible
- **Contingency:** Rankings typically recover within 2-4 weeks
